Codex 为什么不只是聊天机器人？

我不写代码，也开始理解 AI Agent 了

不是因为它“更高级”。

而是它让我第一次感觉到：AI 不只是在回答我，也可以和我一起把事情往前推。

Day2｜Codex 不是另一个聊天框
02
先抛一个问题

我一开始也很疑惑

我又不是程序员，真的需要学 AI Agent 吗？
🫠不写代码
💻不做计算机工作
🤔这个“更高级的 AI”
和我有什么关系？
普通人学 AI，先从真实疑惑开始
03
我现在的理解

ChatGPT 和 Codex 的差别

一个更像陪我想，一个更像陪我做

💬 ChatGPT

陪你想
解释概念
给建议
帮你把话说明白

🤖 AI Agent / Codex

进到任务现场
读文件 / 看路径
拆步骤
执行和验证
不是谁更高级，而是协作方式不同
04
一个很小的例子

WPS 云文档上传失败

从临时清理，到发现路径问题
我学到的不是清理技巧，而是：模糊问题要靠追问变清楚
05
这件小事让我意识到

AI Agent 不只适合
“大项目”

也适合那些碎、烦、说不清的问题

它的价值不是“替我一键解决”。

而是陪我把问题拆开

先判断原因，再决定下一步，最后去验证结果。

AI Agent 的价值，有时就在这些说不清的小麻烦里
06
我真正想记住的是

AI 时代重要的能力

可能不是人人都会写代码
✨ 发现需求
✨ 说清需求
✨ 判断结果
✨ 决定方向
工具会越来越强，人更要练判断
07
Day2 的结论

文科生不一定要会写代码

但要学会和会执行的 AI 协作

我还在 0 基础阶段，也没完全想明白。

但我开始觉得：Codex 不只是一个聊天机器人。

它更像一种新的协作方式
