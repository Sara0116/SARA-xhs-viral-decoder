# 小红书热点搜索工具搭建路线

## 先定目标

这个工具不是为了“抓全网数据”，而是为了服务账号和工作：

- 每天快速发现小红书上和 AI、内容运营、达人营销相关的热门词
- 判断哪些词值得写成笔记
- 沉淀自己的选题库、案例库、标题库
- 最后能形成一个可展示的 AI 内容运营作品集

一句话定位：

> 给广告/内容从业者用的“小红书 AI 选题雷达”。

## 不建议一开始做什么

不建议一上来做自动爬虫。

原因：

- 技术门槛高，容易卡在登录、验证码、反爬、接口变化
- 数据清洗比想象中麻烦
- 容易踩平台规则和客户保密边界
- 对目前目标来说，投入产出比不高

更适合的路线是：

> 手工采集 + 表格结构化 + AI 分析总结 + 后续再半自动化。

## 第 0 阶段：先做一个手工版

工具形态：Excel / 飞书多维表格 / Notion 表格都可以。

建议先用表格，字段如下：

| 字段 | 说明 |
| --- | --- |
| 日期 | 采集日期 |
| 来源 | 小红书搜索、搜索联想、蒲公英、聚光、第三方工具等 |
| 关键词 | 看到的热点词 |
| 所属赛道 | AI 工具、职场、内容运营、达人营销、广告乙方等 |
| 热度信号 | 搜索量、互动高、最近多次出现、平台推荐等 |
| 相关笔记标题 | 记录 3-5 个高互动标题 |
| 内容角度 | 笔记主要在讲什么 |
| 适不适合我写 | 适合 / 一般 / 不适合 |
| 可写选题 | 转成自己的选题 |
| 备注 | 风险、灵感、案例 |

每天只采 10-20 个词，不追求全。

## 第 1 阶段：建立固定采集路径

每天固定看 4 个地方：

1. 小红书 App 搜索框
   - 输入“AI”“Codex”“AI 工作流”“内容运营”“达人营销”等种子词
   - 记录搜索联想词
   - 看搜索结果前排笔记标题、封面、评论区高频问题

2. 小红书站内话题页
   - 看相关话题下最近高互动笔记
   - 重点记录标题结构和评论区需求

3. 广告/商业平台
   - 如果有权限，可以看蒲公英、聚光等平台里的内容关键词、行业趋势、达人内容形态
   - 只记录公开或自己有权限查看的信息，不记录客户敏感数据

4. 第三方数据工具
   - 如果公司已有千瓜、灰豚、新榜等工具，可以用来辅助判断趋势
   - 暂时不用自己买昂贵会员，先用公司已有资源或免费试用

## 第 2 阶段：让 AI 帮你分析

每次采集完，把表格里的关键词和笔记标题丢给 AI，让它输出：

- 今天值得关注的 5 个关键词
- 每个关键词背后的用户需求
- 哪些适合“广告乙方学 AI”账号写
- 每个关键词可延展的 3 个选题
- 哪些词只是热闹但不适合追

可以用这个提示词：

```text
你是一个小红书 AI 内容运营助手。
我会给你一批今天采集到的小红书关键词、热门标题和笔记观察。
请帮我从“广告公司达人营销策划，正在学习 AI 工作流”的账号定位出发，筛选出最值得写的选题。

请输出：
1. 今日最值得关注的 5 个关键词
2. 每个关键词背后的用户需求
3. 是否适合我的账号，理由是什么
4. 可以写成的小红书标题 3 个
5. 正文角度建议
6. 是否存在蹭热点过度、专业度不足或泄密风险
```

## 第 3 阶段：做成半自动工具

等手工版连续跑 2-4 周后，再做半自动化。

半自动工具可以包含：

- 一个表格：存关键词、标题、链接、判断结果
- 一个分析按钮：把当天数据交给 AI 总结
- 一个选题输出页：自动生成今日可写选题
- 一个周报页：总结本周热点变化和账号选题机会

对 0 技术基础来说，优先顺序是：

1. 飞书多维表格 + 手工录入
2. 飞书/Notion + AI 总结
3. 本地 Excel + Codex 帮你分析
4. 以后再考虑网页小工具或自动化脚本

## 第 4 阶段：再考虑技术版

当你已经知道自己每天要看什么字段、怎么判断热点、怎么产出选题之后，再考虑技术实现。

技术版可以做成：

- 一个本地网页
- 左侧输入关键词和笔记观察
- 右侧输出热点评分、选题建议、标题建议
- 支持导出 CSV / Markdown

核心功能不需要复杂：

- 关键词管理
- 热点评分
- AI 分析
- 选题生成
- 周报生成

不要一开始追求“自动抓小红书全站数据”。先让工具变成你每天真的会用的工作台。

## V2：Excel 导入版要注意什么

如果可以从千瓜、蒲公英或公司已有数据工具下载 Excel，V2 版本就不需要爬虫，应该做成“Excel 导入 + 字段标准化 + 选题分析”的工具。

核心原则：

- 工具只处理你有权限下载和使用的数据。
- 原始 Excel 只作为内部分析材料，不直接公开展示。
- 对外发布内容时，只讲趋势判断、方法总结和匿名案例，不贴敏感数据截图。
- 不把客户名、品牌预算、达人报价、联系方式、后台预估数据等直接发给公开 AI 工具。

### 1. 先统一字段，不要被不同平台格式拖住

千瓜、蒲公英、聚光或第三方工具导出的 Excel 字段可能不一样。V2 工具里要先建立一套自己的“标准字段”。

建议标准字段：

| 标准字段 | 说明 |
| --- | --- |
| source_platform | 数据来源：千瓜 / 蒲公英 / 聚光 / 手工 |
| export_date | 导出日期 |
| keyword | 搜索词 / 话题词 / 行业词 |
| note_title | 笔记标题 |
| note_url | 笔记链接 |
| author_name | 达人昵称，公开输出时默认脱敏 |
| author_id | 达人 ID，内部去重用 |
| category | 行业 / 垂类 |
| publish_date | 发布时间 |
| likes | 点赞 |
| collects | 收藏 |
| comments | 评论 |
| engagement | 互动量 |
| fans | 粉丝数 |
| note_type | 图文 / 视频 / 直播切片等 |
| commercial_tag | 是否商业笔记 / 是否报备 |
| estimated_data | 预估阅读、报价、CPM 等敏感字段，默认内部使用 |
| raw_file_name | 原始文件名，方便追溯 |

不同平台字段先进“字段映射表”，再转成统一格式。不要在分析逻辑里直接写死“第几列是什么”。

### 2. 做好去重

Excel 多次下载很容易重复。

建议去重规则：

- 优先用笔记链接去重。
- 没有链接时，用“标题 + 达人昵称 + 发布时间”去重。
- 达人维度用“达人 ID”优先，昵称只做辅助，因为昵称可能会变。

V2 工具最好保留“首次出现日期”和“最近出现日期”，这样才能判断一个词是不是持续升温。

### 3. 区分 3 类数据

不要把所有 Excel 混成一锅。

建议分成：

- 热点词数据：关键词、搜索词、话题词、热度趋势。
- 内容样本数据：笔记标题、链接、发布时间、互动量、评论。
- 达人数据：达人昵称、粉丝量、垂类、报价、合作数据。

做小红书选题时，最常用的是“热点词 + 内容样本”。达人数据更适合做投放策略，不一定要放进公开账号内容里。

### 4. 评分逻辑要可解释

不要只给一个“推荐指数”，要能解释为什么推荐。

建议评分维度：

| 维度 | 分数 | 说明 |
| --- | --- | --- |
| 热度 | 0-2 | 互动量、搜索热度或榜单位置 |
| 增长 | 0-2 | 最近是否明显增加 |
| 相关性 | 0-2 | 是否贴合 AI、内容运营、达人营销 |
| 可写性 | 0-2 | 你是否有真实工作经验可写 |
| 差异化 | 0-2 | 是否能写出广告乙方视角 |
| 风险扣分 | -3 到 0 | 是否涉及客户、报价、内部数据、过度蹭热点 |

输出时要显示：

- 推荐分
- 推荐理由
- 风险提示
- 可写标题
- 正文角度

### 5. 敏感字段要默认脱敏

这些字段默认不要出现在可复制报告里：

- 客户名
- 品牌名
- 达人联系方式
- 达人报价
- 项目预算
- 预估曝光 / 预估阅读 / 预估投放金额
- 合作收入
- 内部备注

如果要写公开笔记，改成：

- 某美妆品牌
- 某腰部达人
- 某个 3C 类目项目
- 一份达人 brief
- 一次竞品内容观察

### 6. 不要把 Excel 原文直接丢给公开 AI

可以先让 V2 工具做本地脱敏：

- 删除品牌名、达人名、联系方式、报价
- 只保留关键词、标题结构、互动区间、内容角度
- 把具体数值改成区间，比如“互动 1000-3000”

脱敏后再交给 AI 总结，更安全。

### 7. V2 的功能优先级

第一版不用做太复杂，按这个顺序：

1. 上传 Excel
2. 选择数据来源
3. 字段自动识别 / 手动映射
4. 数据预览
5. 去重
6. 敏感字段脱敏
7. 热点词归类和评分
8. 生成今日选题
9. 导出选题 CSV / Markdown
10. 生成周报

不要一开始做账号登录、自动定时抓取、多平台接口。先把 Excel 处理流程跑顺。

### 8. V2 可以输出什么

适合输出：

- 今日高潜关键词 Top 10
- 本周上升选题
- AI / 内容运营 / 达人营销分类热点
- 适合自己账号写的选题
- 不建议追的热点及原因
- 小红书标题结构总结
- 每周内容计划

不适合输出：

- 原始达人报价表
- 原始品牌投放数据
- 原始后台截图
- 未脱敏的客户或达人数据

## V2 内置防呆规则

注意事项不要都靠人工记。V2 应该把这些规则直接做进工具里，让工具默认帮你拦截、脱敏、提醒。

### 自动做，不需要人工判断

这些可以直接写进工具逻辑：

1. 自动字段识别
   - 识别“达人昵称、品牌、报价、联系方式、预算、预估阅读、预估曝光、合作金额”等敏感字段。
   - 识别“标题、关键词、链接、发布时间、点赞、收藏、评论”等可分析字段。

2. 自动去重
   - 优先用链接去重。
   - 没有链接时，用“标题 + 达人昵称 + 发布时间”去重。
   - 输出去重前后数量。

3. 自动脱敏
   - 达人昵称改成“达人 A / 达人 B”。
   - 品牌名改成“某品牌 / 某美妆品牌 / 某 3C 品牌”。
   - 报价、预算、联系方式直接从公开报告里删除。
   - 具体互动量可以改成区间，比如“1000-3000”。

4. 自动风险扫描
   - 如果一行里出现手机号、微信号、邮箱、报价、预算、客户名等，标记为“高风险”。
   - 高风险内容不允许进入“公开笔记草稿”。

5. 自动生成两个版本
   - 内部分析版：保留更多字段，只存在本地。
   - 公开选题版：默认脱敏，只保留可公开表达的趋势和选题。

6. 自动生成免责声明提醒
   - 报告底部提示：数据来自授权下载文件，仅用于内部选题分析；公开内容需脱敏处理。

### 需要你点确认

这些最好不要完全自动，因为会影响内容判断：

- 是否保留某个品牌所属行业，比如“美妆 / 3C / 食品饮料”。
- 某条案例是否能写成公开笔记。
- 某个热点是否和你的账号定位匹配。
- 是否把“达人营销”工作流写得更细。
- 是否把某个选题放进当周发布计划。

工具可以给建议，但最后要让你点“可公开 / 仅内部 / 删除”。

### 直接禁止导出

这些内容默认不允许出现在公开导出结果里：

- 手机号
- 微信号
- 邮箱
- 达人报价
- 品牌预算
- 项目预算
- 客户全称
- 真实达人名单
- 后台截图里的原始数据
- 平台声明为不可公开传播的预估数据

即使你手动点导出，工具也应该弹出提醒，并要求先脱敏。

### V2 页面建议

V2 工具可以做成 4 个页面：

1. 上传页
   - 上传千瓜 / 蒲公英 / 聚光 Excel。
   - 选择数据来源。
   - 显示识别到的字段。

2. 清洗页
   - 自动去重。
   - 自动识别敏感字段。
   - 自动脱敏。
   - 显示高风险行。

3. 分析页
   - 热点词排名。
   - 标题结构总结。
   - 内容角度归类。
   - AI / 内容运营 / 达人营销相关性评分。

4. 输出页
   - 今日可写选题。
   - 本周趋势周报。
   - 公开版 Markdown。
   - 内部版 CSV。

### 最省人工的工作流

理想情况下，你每天只做 4 步：

1. 从千瓜 / 蒲公英下载 Excel。
2. 上传到 V2 工具。
3. 看工具自动标出的“推荐写 / 可观察 / 不建议写”。
4. 从公开版选题里挑 1 条写笔记。

人工不要再做复制粘贴、去重、手动删敏感信息、逐条算分这些事。

## V2 进阶：热门视频不只用来找选题

如果你已经能从千瓜、蒲公英或其他工具导出热门视频数据，V2 不应该只做“选题推荐”，还应该做“爆款内容拆解”。

可以从 7 个方向提炼可复用资产：

### 1. 开头钩子

分析热门视频前 3 秒或标题里的吸引点：

- 痛点型：我终于知道为什么……
- 反常识型：原来不是你不会，而是……
- 结果型：我用 AI 把……缩短到……
- 身份型：广告乙方 / 内容运营 / 0 基础用户……
- 场景型：客户临时改 brief 的时候……

V2 输出：

- 高频开头句式
- 可复用开头模板
- 适合你账号的改写版本

### 2. 标题结构

不要只看标题关键词，要拆标题公式。

常见结构：

- 身份 + 场景 + 结果
- 问题 + 解决动作
- 工具 + 具体任务
- 踩坑 + 复盘
- 数字 + 方法

V2 输出：

- 本周高频标题公式
- 每个公式的代表案例
- 适合“广告乙方学 AI”的标题改写

### 3. 视频脚本结构

如果导出的数据里有正文、文案、字幕或你能补充视频摘要，可以拆脚本结构。

建议结构标签：

- 开场：怎么抓注意力
- 痛点：用户卡在哪里
- 过程：怎么一步步做
- 结果：有什么变化
- 反思：哪里还不靠谱
- 互动：怎么引导评论

V2 输出：

- 热门视频常用脚本骨架
- 可套用的脚本模板
- 哪些结构适合图文，哪些适合视频

### 4. 封面和视觉信息

热门视频常常赢在封面第一眼。

可分析：

- 封面主标题字数
- 是否使用强情绪词
- 是否有真人出镜
- 是否有屏幕截图 / 表格 / 工具界面
- 是否有对比结构，比如“以前 vs 现在”
- 色块、箭头、框选、圈重点等视觉元素

V2 输出：

- 封面文字模板
- 封面信息密度建议
- 适合你账号的封面方向

### 5. 评论区需求

如果能导出或手动补充热门评论，价值很高。

评论区可以拆：

- 用户哪里不懂
- 用户想要模板还是教程
- 用户质疑点是什么
- 用户想看下一篇什么
- 是否有人问工具、价格、步骤、案例

V2 输出：

- 用户高频问题
- 下一篇选题
- 评论区回复模板
- 可做成系列的内容线

### 6. 达人表达方式

热门视频还可以拆达人怎么说话。

可分析：

- 语气：朋友聊天 / 老师讲课 / 职场吐槽 / 干货清单
- 人设：新手记录 / 专家建议 / 乙方吐槽 / 工具测评
- 信息密度：轻分享 / 中度教程 / 重度方法论
- 情绪：焦虑、松弛、吐槽、陪伴、效率感

V2 输出：

- 表达风格标签
- 哪些风格适合你
- 哪些风格不适合你

### 7. 商业和职业价值

这部分对你未来转 AI 内容运营很重要。

不要只问“这条热视频能不能仿”，还要问：

- 它服务了什么用户需求？
- 它能不能形成系列？
- 它背后有没有可复用工作流？
- 它能不能沉淀成作品集？
- 它是否能证明你具备内容运营能力？

V2 输出：

- 可复用内容资产
- 可沉淀为作品集的案例
- 可扩展成系列的主题
- 对 AI 内容运营岗位有价值的能力证明

## 热门视频分析字段

V2 可以新增这些标准字段：

| 字段 | 说明 |
| --- | --- |
| video_title | 视频标题 |
| cover_text | 封面文字 |

## 热门视频输出结果

V2 不只输出“今日选题”，还应该输出：

- 爆款开头库

- 封面文字库

最有价值的不是复制热门视频，而是把热门视频拆成“可复用结构”，再换成你的真实工作场景。

## V2 的关键：输出给 Codex 继续使用

V2 的最终产物不应该只是给人看的报告，而应该是 Codex 能继续读取、理解、调用的“内容素材库”。

1. `hotspot_report_YYYY-MM-DD.md`

2. `codex_content_brief_YYYY-MM-DD.md`

### 最终工作流

理想流程是：

1. 下载千瓜 / 蒲公英 Excel。
2. 上传到 V2 工具。
3. 工具自动清洗、脱敏、去重、拆解热门视频。
4. 工具生成 `codex_content_brief_YYYY-MM-DD.md`。
5. 让 Codex 读取这个 brief。
6. Codex 生成下一篇小红书笔记、封面文案、视频脚本和话题词。

这样你的工具就不是单点提效，而是完整内容流水线的一部分。

## 热点评分规则

可以先用人工评分，满分 10 分：

| 维度 | 分数 | 判断方式 |
| --- | --- | --- |
| 热度 | 0-2 | 最近是否频繁出现 |
| 增长 | 0-2 | 是否是新冒出来的词 |
| 相关性 | 0-2 | 是否贴合 AI、内容运营、达人营销 |
| 可写性 | 0-2 | 你是否有真实经历可写 |
| 差异化 | 0-2 | 你能不能写出乙方策划视角 |

总分 8 分以上：优先写。

总分 5-7 分：放进备选。

总分 4 分以下：只记录，不追。

## 每日使用流程

每天 20 分钟即可：

1. 搜 3-5 个种子词
2. 记录 10-20 个相关关键词和标题
3. 用评分规则筛一遍
4. 让 AI 生成 5 个可写选题
5. 选 1 个写成当天笔记

## 每周复盘流程

每周做一次：

- 本周出现次数最多的关键词
- 本周最适合自己账号的 5 个选题
- 哪些热点不适合追
- 哪些选题可以做成系列
- 下一周的 3 篇内容计划

## 合规边界

- 不采集用户隐私信息
- 不记录客户、品牌、达人、报价等敏感信息
- 不用爬虫批量抓取登录态数据
- 不把公司内部资料直接喂给公开 AI 工具
- 如果使用官方商业平台或第三方数据工具，只记录自己有权限使用的数据
